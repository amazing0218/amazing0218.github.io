Java 基础之整型

发表于 2014-04-05 | 分类于 Java 基础系列 | | 本文总阅读量 次

简介

Java语言中包含byte,short, char, int, long 5种原生的整型， 每种类型都有对应的包装类， 这些包装类提供了一些实用方法，用于操作这些原生类型。如下为5种整型的字节长度及表示范围：

整型 字节数 表示范围
byte 1 -2^7 ~ 2^7-1
char 2 0 ~ 2^16-1
short 2 -2^15 ~ 2^15-1
int 4 -2^31 ~ 2^31-1
long 8 -2^63 ~ 2^63-1

从上表能够看出，char比较特殊，是无符号类型，Java中的字符，采用的是Unicode编码，具体详情可参考Java基础之字符)，其他整型都是有符号类型，从计算机组成原理的课程可以知道，计算机一般都是以补码形式来存储整型的。

所有整型类都继承了Number抽象类，实现对应的comparable接口，都有一个唯一的不可变的成员变量value来存储对应的原生类型，下面以Integer为例，介绍一下整型的相关方法，其中一些方法是完全位运算实现，非常有意思，更多关于位运算的例子可参考Bit Twiddling Hacks

hashcode 和 equals

几类整型的的equals都是根据其原始作为比较的，即对象类型相同且value相同，则返回true, 否则返回false。 Byte, Character, Short, Integer 都以对应的int值作为hashcode, Long则是以高32位与低32位的异或结果作为hashcode。

总结

整数位运算非常高效，但是那些方法有实际使用价值么，难道只是为了出面试题，为难面试人员，还是只是为了有趣？ 答案肯定是NO，例如bitCount就有一些实际应用，比如两个比特串的海明距离，即两个比特串对应位置不相同的数量，这个可以先异或，然后bitCount, 而海明距离又是许多算法(例如simhash算法)衡量两个bit串相似性的度量工具。此外，在资源受限的情况下，算法使用高效的位运算，也有可能是唯一选择。
